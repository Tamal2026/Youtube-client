Discuss the scope of var, let, and const

(1)...var : var has function-level scope. It is not block-scoped like let and const. This means that var variables are accessible throughout the entire function in which they are declared. If a var variable is declared inside a block (e.g., an if statement or a for loop), it is still accessible outside of that block. var variables are also hoisted, which means their declarations are moved to the top of the enclosing function or global scope. However, their initializations remain in place.

(2)...let : let has block-level scope, meaning it is accessible only within the block (enclosed by curly braces) in which it is declared. Variables declared with let are not hoisted, and they are not accessible before their declaration point in the code.

(3)...const : const: const also has block-level scope, similar to let. Variables declared with const are not hoisted. Unlike let, const variables must be initialized when declared, and their values cannot be reassigned once they are set. However, for objects and arrays, the properties or elements can be modified.

The use cases of null and undefined..

Undefined: undefined is a primitive value
in JavaScript that represents the absence of a value or the uninitialized state of a variable. It is often automatically assigned to a variable when it is declared but not initialized with a value. undefined is commonly used when a function doesn't return a value explicitly, or when an expected object property or array element is missing.

Null: null is a special value in JavaScript that represents the intentional absence of any object value. It is often explicitly assigned to a variable or property by a programmer to indicate that the variable should have no object reference. Unlike undefined, null is an intentional indication of missing or absent value, and it must be assigned explicitly.

A REST API is an API that conforms to the design principles of the REST, or representational state transfer architectural style.
